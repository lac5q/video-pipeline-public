**scripts/batch-produce.sh** -- Refactor to produce BOTH video types per order.

Key changes:

1. **Integrate asset download and staging into the batch flow.** Currently batch-produce.sh assumes assets and staged mockups already exist. Add the full pipeline steps before video build:
   - Check if workspace exists; if not, download assets via `download-order-assets.sh`
   - Check if mockups exist; if not, generate via `generate-mockups.js`
   - Check if staged (v11_*.png without .raw backup); if not, stage via `stage-products.sh`

2. **Dual video build.** Replace the single build step with:
```bash
# Detect reaction video
HAS_REACTION=false
for ext in mov mp4 MOV MP4; do
    if [[ -f "${WORKSPACE}/${ORDER_ID}.${ext}" ]] || [[ -f "${WORKSPACE}/reaction.${ext}" ]]; then
        HAS_REACTION=true
        break
    fi
done

# Build standard reel (always)
echo "  Building standard reel..."
if "${PIPELINE_ROOT}/scripts/build-video.sh" "$BRAND_CONFIG" "$WORKSPACE" "$PRODUCTS_CONFIG" "reels"; then
    echo "  Standard reel complete: ${ORDER_ID}_reel.mp4"
else
    echo "  ERROR: Standard reel build failed"
    FAILED=$((FAILED + 1))
    RESULTS+=("${ORDER_ID}: FAILED (reel build error)")
    continue
fi

# Build UGC reel (only if reaction video exists)
if [[ "$HAS_REACTION" = true ]]; then
    echo "  Building UGC reel..."
    if "${PIPELINE_ROOT}/scripts/build-video.sh" "$BRAND_CONFIG" "$WORKSPACE" "$PRODUCTS_CONFIG" "ugc"; then
        echo "  UGC reel complete: ${ORDER_ID}_ugc.mp4"
    else
        echo "  WARNING: UGC reel build failed (standard reel still available)"
    fi
else
    echo "  No reaction video — skipping UGC reel"
fi
```

```bash
echo "  [SKIP] Order ${ORDER_ID} failed — continuing batch"
```
