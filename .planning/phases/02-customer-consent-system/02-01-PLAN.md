  <action>
    Create `scripts/approve-consent-candidates.js` -- the Phase 2 batch approval CLI for consent (NOT production).

    Key difference from existing approve-orders.js: This script sets `consent_status = 'pending'` to mark orders as ready to receive consent emails. It does NOT touch `production_status`.

    Usage: `node scripts/approve-consent-candidates.js [OPTIONS]`

    Options:
    - `--brand SLUG` -- filter by brand
    - `--limit N` -- max candidates (default 20)
    - `--min-score N` -- minimum score threshold (default 30)
    - `--list` -- show all candidates in a summary table first (list mode), then optionally drill into individual
    - `--help` -- show help

    Flow:
    1. Query orders WHERE `consent_status = 'pre_approved'` AND `production_status = 'pending'` (candidates that have not yet been approved for consent emails)
    2. Score and rank using `require('../lib/scorer').rankOrders(candidates)`
    3. If `--list` flag: show a compact table with columns: Rank, Order ID, Brand, Score, Reaction (has_reaction_video yes/no), Customer Name (or "?" if null), # Items, OMS URL (truncated)
    4. Then enter interactive mode (same as current approve-orders.js pattern using readline):
       - Display each candidate with: illustration preview URL (construct from `oms_url` field), customer name + order ID, brand + product type (from tags), reaction video availability, number of people (from `description` field), order date
       - Actions: [a]pprove (sets consent_status='pending'), [s]kip, [r]eject (sets consent_status='denied'), [v]iew details, [q]uit
    5. On approve: `require('../lib/consent').updateConsent(orderId, brand, 'pending', 'Luis approved for consent email')`
    6. On reject: `require('../lib/consent').updateConsent(orderId, brand, 'denied', 'Luis rejected for consent')`
    7. Show summary at end: approved, rejected, skipped counts

    Per CONTEXT.md locked decisions, each candidate display must show:
    - Illustration preview (OMS URL or image URL)
    - Customer name + order ID
    - Brand + product type
    - Whether reaction video is available
    - Number of people in the illustration
    - Order price (if available, else "-")
    - Number of items (if available, else "-")
    - Date of order

    Use `require('dotenv').config()` at top. Use `require('../lib/consent')` for DB operations. Use `require('../lib/scorer')` for ranking.
  </action>
