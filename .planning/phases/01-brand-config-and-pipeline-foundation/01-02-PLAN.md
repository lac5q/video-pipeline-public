**produce-video.sh** -- Create the CLI entrypoint at the repo root. This script:

1. Parses CLI arguments: `--brand SLUG --order ORDER_ID` (both required)
2. Auto-detects `PIPELINE_ROOT` using: `PIPELINE_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"`
3. Validates the brand config exists: `${PIPELINE_ROOT}/brands/${BRAND}.json`
4. Validates products.json exists: `${PIPELINE_ROOT}/products.json`
5. Validates required env vars are set: GEMINI_API_KEY (warn if missing, don't fail -- staging may already be done)
6. Loads brand config values via jq (see RESEARCH.md Pattern 3 for exact jq commands):
   - BRAND_NAME, BRAND_SLUG, BRAND_BG, BRAND_ACCENT, HOOK_ACCENT, LABEL_COLOR
   - LOGO_FILE, LOGO_WIDTH, CTA_LINE1, CTA_URL, CTA_TAGLINE
   - SHOWCASE_KEY (product_showcase_order), REACTION_LABEL, FONT
7. Creates per-order workspace: `${PIPELINE_ROOT}/orders/${BRAND}/${ORDER_ID}/`
   - Create subdirectories: mockups/, photos/, exports/
8. Exports all config values as environment variables for subscripts
9. Calls `scripts/build-video.sh "${BRAND_CONFIG}" "${WORKSPACE}" "${PRODUCTS_CONFIG}"` when assets are ready
10. After build, calls `scripts/verify-video.sh` on the output
11. Reports success/failure with output file path

CRITICAL path rules:
- EVERY file reference MUST use `${PIPELINE_ROOT}/...` or be relative to workspace
- ZERO hardcoded `/Users/lcalderon/` paths
- Logo resolved as: `${PIPELINE_ROOT}/brands/assets/logos/${LOGO_FILE}`
- Music resolved as: `${MUSIC_DIR:-${PIPELINE_ROOT}/brands/assets/music}/FILENAME`

Include `--skip-build` flag (for when you only want to set up workspace, not build video)
Include `--help` flag showing usage.

The script should detect if reaction video exists in workspace and set HAS_REACTION=true/false (used by build script to decide UGC vs reels-only).

**scripts/verify-video.sh** -- Quality gate script:

1. Takes video file path as argument
2. Uses ffprobe to check: width=1080, height=1920, codec=h264, fps~30
3. Checks file size > 0
4. Reports PASS/FAIL for each check
5. Exits 0 on all pass, 1 on any fail

Use the verify_video() function from RESEARCH.md code examples as a starting point.

Both scripts MUST start with `#!/bin/bash` and `set -euo pipefail`.
Both scripts MUST be executable (chmod +x).
